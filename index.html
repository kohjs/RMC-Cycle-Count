üñ•Ô∏è Backend System Processes

The following processes are automated and occur on the backend system before the operator uses the CK3 device:

üìÖ Automated Scheduling

  • Generate 2-3 customers per month
  • Group pallets by customer
  • Sort by age (older pallet numbers first)
  • Select 5% sample of aging pallets
  • Create cycle count ID and list

üì¶ Bundle Sampling

  • Select sample bundles from sampled pallets
  • Record Pallet-Lot-Bundle-Location
  • Store customer info & reject types
  • Generate master sample list

üíæ Database Recording

  • Record full cycle count start date
  • Link to existing RMC data
  • Store complete cycle count data
  • Set status to "Pending"

üìß Email Notifications

  • Send initial alert to RMC team
  • Include count ID, customer, due date
  • List pallet count and bundle count
  • Send weekly reminders until completed

üîÑ Internal Background Refresh

Scheduled weekly or when scrap list is generated. System checks available bundles in aging pallets, replaces pallets if insufficient bundles, and replaces sample bundles if they appear in scrap list.
